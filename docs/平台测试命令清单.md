### 1. 网络连通

### 2. 区块链正常出块
```bash
C=$(docker ps --format '{{.Names}}' | head -n 1)
docker exec "$C" bash -lc 'bitcoin-cli -conf=/data/bitcoin/bitcoin.conf -datadir=/data/bitcoin getblockchaininfo'
```

### 3. 节点正常同步
```bash
docker exec "$C" bash -lc 'bitcoin-cli -conf=/data/bitcoin/bitcoin.conf -datadir=/data/bitcoin getblockchaininfo'
# 看 blocks == headers，且 initialblockdownload=false
```

### 4. 不定期自动发送交易（简查）
```bash
docker exec "$C" bash -lc 'bitcoin-cli -conf=/data/bitcoin/bitcoin.conf -datadir=/data/bitcoin getmempoolinfo'
```

### 5. 攻击策略（7 选 1~2）

```bash
# 示例A：限速60秒

# 示例B：丢包60秒
docker exec "$C" tc qdisc add dev eth0 root netem loss 5%
sleep 60

```

## 联盟链（Fabric）测试命令

### 登录
```bash
ssh ubuntu@119.91.51.36 -p 22
# 密码: 592087469hW!@
```

### 1. 网络连通
```bash
hostname
uptime
docker ps --format 'table {{.Names}}\t{{.Status}}' | head -30
```

### 2. 区块链正常出块
```bash
docker logs --tail 200 peer0.org1.fabricvirtnet.com | grep -Ei 'Committed block|Created block|Received block' | tail -20
```

### 3. 节点正常同步
```bash
docker exec fabric-cli-12as peer version
docker exec fabric-cli-12as bash -lc 'peer channel list'
```

### 4. 不定期自动发送交易（简查）
```bash
docker logs --tail 300 peer0.org1.fabricvirtnet.com | grep -Ei 'endorse|invoke|commit' | tail -20
```

### 5. 攻击策略（7 选 1~2）
```bash
P=peer0.org1.fabricvirtnet.com

# 示例A：延迟+抖动60秒
docker exec "$P" tc qdisc add dev eth0 root netem delay 200ms 50ms
sleep 60
docker exec "$P" tc qdisc del dev eth0 root

# 示例B：丢包60秒
docker exec "$P" tc qdisc add dev eth0 root netem loss 5%
sleep 60
docker exec "$P" tc qdisc del dev eth0 root
```

### 8. 合约自动部署检查
```bash
docker ps --format '{{.Names}} {{.Image}}' | grep -Ei 'dev-peer0\.org1|dev-peer0\.org2|chaincode'
```
