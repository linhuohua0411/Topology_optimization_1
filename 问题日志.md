## Critical（高风险，可能影响结论有效性）

### 1) MLE 验证阶段未进行时序递推，评估逻辑与“预测”叙述不一致

**证据：**

- 在 4.1.1 中 `A_pred` 初始化后未在时间循环内更新，导致每个时刻都在拿同一张初始图与未来快照比较。

```python
A_pred = snapshots[0].copy()
...
for t in range(1, min(len(snapshots), 11)):
    A_true = snapshots[t]
    binary_pred = (A_pred > 0.5).astype(float)
    binary_true = (A_true > 0.5).astype(float)
```

**影响：**

- 当前 `edge_accuracy/degree/path/cluster` 指标更接近“初始图与未来图差异”而非“模型时序预测误差”。
- 4.1.1 的“参数估计有效性”证据强度被削弱，可能被审稿人质疑评估定义与实现不一致。

**建议：**

- 明确二选一：  
  1) 若目标是 one-step 预测：每轮用动力学方程更新 `A_pred`；  
  2) 若目标是静态偏移分析：修改论文与代码注释，避免称为预测准确率。

---

### 2) 优化目标与梯度方向不完全一致（目标是 R，总梯度近似只用 R_s）

**证据：**

- 总目标定义为 `R = w1*R_s + w2*R_c + w3*R_r`。
- 梯度近似函数注释写明“仅对 `R_s` 做差分以加速”，并在计算梯度时仅用 `w1`。

```python
def compute_gradient_R(...):
    """鲁棒性梯度近似 (仅对 R_s 做差分以加速)。"""
...
w1 = weights[0]
...
g = w1 * (r_s_pert - r_s_base) / epsilon
```

**影响：**

- 训练信号可能偏向 `R_s`，与论文叙述的“联合优化 `R_s/R_c/R_r`”存在张力。
- 对比实验中出现某些方法 `R_final` 更高而本方法耗时更长，解释难度增大。

**建议：**

- 至少补充消融：`R_s-only gradient` vs `full surrogate gradient`（哪怕是近似的 `R_c/R_r` 增量）。
- 在论文中明确说明该近似及其带来的计算-精度权衡。

## Major（中高风险，影响说服力与公平性）

### 3) 基线对比预算口径不一致，结论存在公平性争议

**证据：**

- Ours: `max_steps=200`；ResiNet: `max_rewires=200`；FPSblo: `max_iters=50`；Static: `max_iters=200`；AttackSim: `max_rewires=100`。
- 不同方法的“迭代含义、每步计算量、停止准则”并不统一。

**影响：**

- “谁更优”结果可能部分来自预算配置差异，而非方法本身。
- 审稿人容易要求补做同时间预算或同边改动预算对比。

**建议：**

- 增加三组公平对比：同时间预算、同边改动预算、同收敛阈值预算。
- 主文结论改为“优于部分基线”，避免“全面优于”表述。

---

### 4) 攻击实验对随机攻击的区分度不足，LCC 指标信息量偏低

**证据：**

- `all_results.json` 中随机攻击下 `baseline_lcc_mean == optimized_lcc_mean`（0.95/0.90/0.85），标准差为 0。
- 现有攻击评价主要使用 `lcc_ratio`，在固定删点比例下易退化为近线性结果。

**影响：**

- 随机攻击场景难显示优化收益，削弱“攻击鲁棒性提升”的完整性。

**建议：**

- 保留 LCC，同时补充全局效率下降率、ASPL 增量、组件数分布等更敏感指标。

---

### 5) 真实网络中连通/非连通情况下路径指标口径不统一，可能产生误读

**证据：**

- 图不连通时用 LCC 内平均路径，连通时用全图平均路径。
- `eth_real_results.json` 中优化前不连通、优化后连通，路径从 `1.70` 到 `1.55`（另一版本文件中出现从 `1.01` 到 `2.77` 的情况也会被同类口径问题触发误读）。

**影响：**

- 路径指标在不同连通状态下不可直接横向比较，结论叙述容易被质疑。

**建议：**

- 同时报告两套口径：全图可达对平均（含不可达处理）与 LCC 口径，避免混用单一数字。

## 当前结论可信度分级

### 可强结论

- 合成拓扑（BA/WS/ER）下优化能稳定提升 `R`。  
- 参数扰动与重复实验显示方法具有一定稳定性。  
- 真实 ETH 场景下 Ours 相比部分基线有收益。

### 可弱结论

- “攻击鲁棒性全面提升”应改为“对定向攻击更明显，随机攻击在现有指标下区分度有限”。  
- “优于主流方法”应改为“优于部分基线，且具备统一鲁棒性建模优势”。

### 暂不主张

- 暂不主张“全面 SOTA”。  
- 暂不主张“已具备强大规模可扩展性”（`N=150` 耗时跃迁显著）。

### 2026-03-23（第4次修改，基于最新终端与结果文件的实验改进方案）

#### A. 当前结果驱动的关键改进方向（按优先级）

##### A-P0（本周必须完成）

1. **修复并重定义 MLE 评估口径（避免指标异常放大）**  
   - 现象：`edge_accuracy` 仍高，但 `degree/path/cluster` 误差显著放大（出现 10+、4+ 量级）。  
   - 为什么要改：  
     - 当前“高边准确率 + 高结构误差”说明评估指标之间可能在度量对象上不一致（离散边命中 vs 连续结构统计），会导致“模型是否有效”结论自相矛盾；  
     - 若不拆分 one-step 与 multi-step，审稿人会质疑该实验是在评估“参数拟合能力”还是“长时预测稳定性”。  
   - 改进动作：  
     - 将 one-step 预测与 multi-step 递推分开报告；  
     - 增加二值阈值敏感性（0.3/0.5/0.7）与连续权重误差（Frobenius/MSE）；  
     - 对 `A_pred` 增加与观测步长一致的 `dt` 缩放，避免离散步长导致漂移。  
   - 验收标准：  
     - one-step 指标稳定；  
     - multi-step 指标随 horizon 单调恶化但不爆炸；  
     - 误差分布图可解释。

2. **统一 ETH 主数据口径并冻结主结果版本**  
   - 现象：`eth_real_results.json` 与 `dual_network_results.json` 的 ETH 基线（边数、R、提升幅度）冲突。  
   - 为什么要改：  
     - 同一网络对象出现两套主结论会直接削弱论文证据一致性，属于高风险可复现实验缺陷；  
     - 不冻结主版本会导致后续图表、表格、结论在不同脚本间反复漂移，最终无法形成可审稿的稳定叙事。  
   - 改进动作：  
     - 明确唯一主数据源（建议统一为 `snapshot_*.json`）；  
     - 固化快照排序、时间窗口、节点筛选规则；  
     - 在结果 JSON 写入 `data_source + snapshot_count + hash/时间范围` 元信息。  
   - 验收标准：  
     - 单网 ETH 与双网 ETH 在同口径下基线一致（允许数值微小浮动）；  
     - 主文仅保留一套 ETH 主结论。

3. **修正攻击实验叙述口径，补“攻击前状态”报告项**  
   - 现象：随机攻击下 LCC 提升弱；双网 ETH 中存在“基线本身低连通+优化后高连通”混合解释风险。  
   - 为什么要改：  
     - 攻击效果必须建立在可比初始状态上，否则“攻击后更好/更差”可能只是初始连通性差异，而非方法抗攻击能力；  
     - 仅报告 LCC 容易忽略网络碎裂细节，无法解释为何路径与连通性指标有时同步、有时背离。  
   - 改进动作：  
     - 每组攻击实验前新增：`connected/lcc_ratio/n_components/global_efficiency`；  
     - 攻击后同时报告：LCC 比例、组件数、效率下降率、LCC 绝对节点数。  
   - 验收标准：  
     - 每条攻击结论均可追溯到“攻击前状态 + 攻击后变化”；  
     - 不再出现“仅凭单一 LCC 数字难解释”的条目。

##### A-P1（1~2 周内完成）

### 2025-03-23（第9次修改，拓扑优化思路与实验逻辑深度分析）

**记录时间：2025-03-23**

#### 一、拓扑优化思路的核心逻辑

当前方法采用**演化动力学 + 自组织动力学**串联：

```
单步流程: 演化 RK4 → 自组织 RK4 → 约束投影
演化: dA/dt = α∇R - βA + σξ
自组织: dA/dt = α_L·F_L(A) + α_G·F_G(A)
  F_L = common/(max_deg) - λ_L·A  (局部三角化)
  F_G = ∇R - λ_G·A                (梯度驱动)
```

目标函数：`R = w1·R_s + w2·R_c + w3·R_r`

---

#### 二、思路与逻辑层面的问题

##### 问题 9.1：优化目标与梯度方向本质脱节

**现象：**

- 目标为 `R = w1·R_s + w2·R_c + w3·R_r`，但 `compute_gradient_R` 仅对 `R_s` 做差分。
- 梯度形式为 `g = w1·(R_s_pert - R_s_base)/ε`，完全忽略 R_c、R_r。

**逻辑后果：**

- R_c（连接鲁棒性，基于全图最短路）和 R_r（恢复鲁棒性，基于度分布）**无直接梯度信号**，仅通过 F_L 的自组织间接影响。
- 实际优化的是「R_s 的梯度方向」，R_c、R_r 的改善是副产物，非主动优化。
- 与论文“联合优化 R_s/R_c/R_r”的表述不一致，易被审稿人质疑目标-实现不匹配。

**建议：**

- 在论文中明确写成“以 R_s 梯度为代理的近似优化”，并补充 R_s-only vs full-gradient 的消融实验，说明近似带来的取舍。

---

##### 问题 9.2：MLE 动力学与优化动力学是两套模型

**现象：**

- MLE 拟合的动力学：`dA = (α·(-β·A) + α_L·F_L + α_G·F_G)·dt`，其中 `F_G = -λ_G·A`（无梯度项）。
- 优化使用的动力学：演化阶段含 `α·∇R`，自组织阶段 `F_G = ∇R - λ_G·A`。

**逻辑后果：**

- MLE 刻画的是**无梯度驱动的被动演化**，优化使用的是**梯度驱动的主动演化**，二者本质不同。
- 优化全程使用 `DEFAULT_PARAMS`，未用 MLE 估计参数，MLE 与优化在代码和逻辑上完全解耦。
- 若论文声称“基于数据驱动的动力学优化”，则存在叙事断裂：拟合的模型并不含优化所需的梯度项。

**建议：**

- 将 MLE 定位为“拓扑演化预测模型验证”，与“基于梯度的拓扑优化”分开叙述；或说明优化采用固定参数、不依赖 MLE 估计。

---

##### 问题 9.3：自组织项 F_L 与目标 R 无显式对应

**现象：**

- `F_L = common/(max_deg) - λ_L·A` 鼓励三角形形成，与 R_clust（聚类系数）在直觉上一致。
- 但 F_L 并非由 ∇R_clust 推导，而是启发式设计。

**逻辑后果：**

- 演化阶段（∇R_s）与自组织阶段（F_L + F_G）的组合缺乏统一的优化理论依据。
- 无法证明该组合收敛到 R 的局部最优，只能视为经验有效的混合策略。

**建议：**

- 在论文中说明 F_L 为启发式自组织项，用于补充局部结构，不宣称其来自 R 的梯度分解。

---

##### 问题 9.4：约束与梯度的处理顺序

**现象：**

- 每步流程：`A_next = constrain(A + dt·dynamics(A, ∇R(A)))`
- 梯度在约束前的 A 上计算，约束投影在步进之后施加。

**逻辑后果：**

- 梯度未考虑 k_max 度约束，可能指向会被约束裁掉的方向。
- 对于高度约束的网络（如区块链 P2P 的度上界），优化效率可能受影响，但不会导致错误结论。

**建议：**

- 可保留现有实现，在论文中简述为“先梯度步进、后约束投影”的两阶段更新，必要时讨论对收敛的影响。

---

##### 问题 9.5：基线与 Ours 的优化目标一致但机制不同

**现象：**

- ResiNet、Static 等基线均以 `compute_R(A)` 为目标，通过贪心试错（边重连）直接优化 R。
- Ours 通过连续动力学的梯度近似优化，梯度仅针对 R_s。

**逻辑后果：**

- 基线是**对 R 的精确贪心优化**，Ours 是**对 R_s 的梯度近似优化**。
- 当 ResiNet/Static 在同时间预算下 R 更高时，可解释为：它们直接优化 R，而 Ours 优化的是 R 的近似代理。

**建议：**

- 论文中强调 Ours 的**动力学可解释性**与**多目标均衡**（通过 F_L、约束等间接影响 R_c、R_r），而非在纯 R 数值上“优于”直接优化 R 的基线。

---

##### 问题 9.6：R_r 的优化机制薄弱

**现象：**

- `R_r = mean(exp(-τ_i/τ_max))`，`τ_i = 1/(1+k_i)`，仅依赖度分布。
- 梯度仅含 R_s，R_s 含 R_degree = exp(-CV)，与度分布相关，但与 R_r 形式不同。

**逻辑后果：**

- R_r 的改善完全依赖 R_degree 的优化和 F_L 的间接作用，缺乏专门针对 R_r 的优化信号。
- 若论文强调“恢复鲁棒性”是重要子目标，需说明当前方法对其的优化是间接的。

**建议：**

- 在方法章节说明 R_r 通过度分布正则化（R_degree）和自组织项间接优化；或补充 R_r 的显式梯度/代理目标。

---

#### 三、拓扑优化思路小结

| 层面         | 当前状态                         | 与 TIFS 的契合度                     |
|--------------|----------------------------------|--------------------------------------|
| 目标定义     | R = w1·R_s + w2·R_c + w3·R_r     | 清晰                                  |
| 梯度实现     | 仅 ∇R_s 近似                     | 与“联合优化”表述不一致，需澄清        |
| MLE–优化关系 | 完全解耦，动力学形式不同         | 需区分“预测模型”与“优化模型”         |
| 自组织 F_L   | 启发式设计，非梯度推导           | 需说明为辅助项，非理论推导            |
| 与基线对比   | 基线直接优化 R，Ours 优化代理    | 结论应强调机制差异，非单纯 R 数值     |

**论文叙事建议：** 将本方法定位为“基于演化动力学与自组织的拓扑优化框架”，明确梯度近似（R_s-only）与启发式自组织项的设计，强调可解释性与多目标均衡，避免与“对 R 的精确优化”直接对标。

### 2026-03-26（第22次修改，各方法当前实验职责整理——精简）

**一、优化方法层（算法做了什么）**

- **Ours（`run_optimization`, full 梯度）**  
  使用 `R=(0.3,0.4,0.3)` 目标做迭代拓扑优化；支持 MLE 参数注入与自适应 `k_max`；允许较大规模结构调整（ETH 场景出现大量 add/modify），是当前主方法。

- **ResiNet（简化版）**  
  基于随机边重连的贪心提升；当前已支持 `allow_disconnected_intermediate` 与 `disconnect_penalty`，可在不连通初态下继续搜索。

- **FPSblo-EP（简化版）**  
  采用 landmark/farthest-point 思路做局部重连；改边规模通常较小，运行快，作为轻量基线。

- **Static（简化版）**  
  依据高度/低度节点进行静态重连，优化度分布与聚类相关结构特征；当前同样支持“中间态可不连通 + 软惩罚”。

- **AttackSim（简化版）**  
  以攻击鲁棒性（LCC）为内层评分进行重连搜索，并与 `R` 软目标联合；用于“攻击导向”对照。
