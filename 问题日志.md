## Critical（高风险，可能影响结论有效性）

### 1) MLE 验证阶段未进行时序递推，评估逻辑与“预测”叙述不一致

**证据：**

- 在 4.1.1 中 `A_pred` 初始化后未在时间循环内更新，导致每个时刻都在拿同一张初始图与未来快照比较。

```python
A_pred = snapshots[0].copy()
...
for t in range(1, min(len(snapshots), 11)):
    A_true = snapshots[t]
    binary_pred = (A_pred > 0.5).astype(float)
    binary_true = (A_true > 0.5).astype(float)
```

**影响：**

- 当前 `edge_accuracy/degree/path/cluster` 指标更接近“初始图与未来图差异”而非“模型时序预测误差”。
- 4.1.1 的“参数估计有效性”证据强度被削弱，可能被审稿人质疑评估定义与实现不一致。

**建议：**

- 明确二选一：  
  1) 若目标是 one-step 预测：每轮用动力学方程更新 `A_pred`；  
  2) 若目标是静态偏移分析：修改论文与代码注释，避免称为预测准确率。

---

### 2) 优化目标与梯度方向不完全一致（目标是 R，总梯度近似只用 R_s）

**证据：**

- 总目标定义为 `R = w1*R_s + w2*R_c + w3*R_r`。
- 梯度近似函数注释写明“仅对 `R_s` 做差分以加速”，并在计算梯度时仅用 `w1`。

```python
def compute_gradient_R(...):
    """鲁棒性梯度近似 (仅对 R_s 做差分以加速)。"""
...
w1 = weights[0]
...
g = w1 * (r_s_pert - r_s_base) / epsilon
```

**影响：**

- 训练信号可能偏向 `R_s`，与论文叙述的“联合优化 `R_s/R_c/R_r`”存在张力。
- 对比实验中出现某些方法 `R_final` 更高而本方法耗时更长，解释难度增大。

**建议：**

- 至少补充消融：`R_s-only gradient` vs `full surrogate gradient`（哪怕是近似的 `R_c/R_r` 增量）。
- 在论文中明确说明该近似及其带来的计算-精度权衡。

## Major（中高风险，影响说服力与公平性）

### 3) 基线对比预算口径不一致，结论存在公平性争议

**证据：**

- Ours: `max_steps=200`；ResiNet: `max_rewires=200`；FPSblo: `max_iters=50`；Static: `max_iters=200`；AttackSim: `max_rewires=100`。
- 不同方法的“迭代含义、每步计算量、停止准则”并不统一。

**影响：**

- “谁更优”结果可能部分来自预算配置差异，而非方法本身。
- 审稿人容易要求补做同时间预算或同边改动预算对比。

**建议：**

- 增加三组公平对比：同时间预算、同边改动预算、同收敛阈值预算。
- 主文结论改为“优于部分基线”，避免“全面优于”表述。

---

### 4) 攻击实验对随机攻击的区分度不足，LCC 指标信息量偏低

**证据：**

- `all_results.json` 中随机攻击下 `baseline_lcc_mean == optimized_lcc_mean`（0.95/0.90/0.85），标准差为 0。
- 现有攻击评价主要使用 `lcc_ratio`，在固定删点比例下易退化为近线性结果。

**影响：**

- 随机攻击场景难显示优化收益，削弱“攻击鲁棒性提升”的完整性。

**建议：**

- 保留 LCC，同时补充全局效率下降率、ASPL 增量、组件数分布等更敏感指标。

---

### 5) 真实网络中连通/非连通情况下路径指标口径不统一，可能产生误读

**证据：**

- 图不连通时用 LCC 内平均路径，连通时用全图平均路径。
- `eth_real_results.json` 中优化前不连通、优化后连通，路径从 `1.70` 到 `1.55`（另一版本文件中出现从 `1.01` 到 `2.77` 的情况也会被同类口径问题触发误读）。

**影响：**

- 路径指标在不同连通状态下不可直接横向比较，结论叙述容易被质疑。

**建议：**

- 同时报告两套口径：全图可达对平均（含不可达处理）与 LCC 口径，避免混用单一数字。

## 当前结论可信度分级

### 可强结论

- 合成拓扑（BA/WS/ER）下优化能稳定提升 `R`。  
- 参数扰动与重复实验显示方法具有一定稳定性。  
- 真实 ETH 场景下 Ours 相比部分基线有收益。

### 可弱结论

- “攻击鲁棒性全面提升”应改为“对定向攻击更明显，随机攻击在现有指标下区分度有限”。  
- “优于主流方法”应改为“优于部分基线，且具备统一鲁棒性建模优势”。

### 暂不主张

- 暂不主张“全面 SOTA”。  
- 暂不主张“已具备强大规模可扩展性”（`N=150` 耗时跃迁显著）。

**建议：**

- 固化“唯一主数据版本”（快照来源、时间范围、采样间隔、节点筛选规则）。

### 2026-03-23（第4次修改，基于最新终端与结果文件的实验改进方案）

#### A. 当前结果驱动的关键改进方向（按优先级）

##### A-P0（本周必须完成）

1. **修复并重定义 MLE 评估口径（避免指标异常放大）**  
   - 现象：`edge_accuracy` 仍高，但 `degree/path/cluster` 误差显著放大（出现 10+、4+ 量级）。  
   - 为什么要改：  
     - 当前“高边准确率 + 高结构误差”说明评估指标之间可能在度量对象上不一致（离散边命中 vs 连续结构统计），会导致“模型是否有效”结论自相矛盾；  
     - 若不拆分 one-step 与 multi-step，审稿人会质疑该实验是在评估“参数拟合能力”还是“长时预测稳定性”。  
   - 改进动作：  
     - 将 one-step 预测与 multi-step 递推分开报告；  
     - 增加二值阈值敏感性（0.3/0.5/0.7）与连续权重误差（Frobenius/MSE）；  
     - 对 `A_pred` 增加与观测步长一致的 `dt` 缩放，避免离散步长导致漂移。  
   - 验收标准：  
     - one-step 指标稳定；  
     - multi-step 指标随 horizon 单调恶化但不爆炸；  
     - 误差分布图可解释。

2. **统一 ETH 主数据口径并冻结主结果版本**  
   - 现象：`eth_real_results.json` 与 `dual_network_results.json` 的 ETH 基线（边数、R、提升幅度）冲突。  
   - 为什么要改：  
     - 同一网络对象出现两套主结论会直接削弱论文证据一致性，属于高风险可复现实验缺陷；  
     - 不冻结主版本会导致后续图表、表格、结论在不同脚本间反复漂移，最终无法形成可审稿的稳定叙事。  
   - 改进动作：  
     - 明确唯一主数据源（建议统一为 `snapshot_*.json`）；  
     - 固化快照排序、时间窗口、节点筛选规则；  
     - 在结果 JSON 写入 `data_source + snapshot_count + hash/时间范围` 元信息。  
   - 验收标准：  
     - 单网 ETH 与双网 ETH 在同口径下基线一致（允许数值微小浮动）；  
     - 主文仅保留一套 ETH 主结论。

3. **修正攻击实验叙述口径，补“攻击前状态”报告项**  
   - 现象：随机攻击下 LCC 提升弱；双网 ETH 中存在“基线本身低连通+优化后高连通”混合解释风险。  
   - 为什么要改：  
     - 攻击效果必须建立在可比初始状态上，否则“攻击后更好/更差”可能只是初始连通性差异，而非方法抗攻击能力；  
     - 仅报告 LCC 容易忽略网络碎裂细节，无法解释为何路径与连通性指标有时同步、有时背离。  
   - 改进动作：  
     - 每组攻击实验前新增：`connected/lcc_ratio/n_components/global_efficiency`；  
     - 攻击后同时报告：LCC 比例、组件数、效率下降率、LCC 绝对节点数。  
   - 验收标准：  
     - 每条攻击结论均可追溯到“攻击前状态 + 攻击后变化”；  
     - 不再出现“仅凭单一 LCC 数字难解释”的条目。

##### A-P1（1~2 周内完成）

1. **公平预算对比实验（4.3 重跑）**  
   - 为什么要改：  
     - 现有结论受到“每种方法预算定义不同”影响，无法排除资源配置造成的排名偏差；  
     - 公平预算是 TIFS 类评审中判断“方法优劣是否成立”的基本前提。  
   - 增加三种预算：同时间预算、同改边预算、同收敛阈值预算。  
   - 输出对照表：`R_final / R_s,R_c,R_r / 时间 / 改边数 / 收敛步数`。  
   - 目标：把“方法效果”与“预算差异”解耦。

2. **可扩展性拐点定位（N=120/130/140）+ 耗时拆解**

3. **DOT 失败模式实验**
