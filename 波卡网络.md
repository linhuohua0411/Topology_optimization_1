# 100 节点虚拟化广域网 Polkadot 部署说明与技术同构性评估

## 1. 文档目标

本文档完整说明以下三件事：

1. 如何在本仓库中构建一个 **10 个 AS（201–210）+ IX 的虚拟化广域网（WAN）**。
2. 如何在该 WAN 上部署 **100 节点规模（10×10）** 的 Polkadot 实验链网络。
3. 该实验网络与真实公链 Polkadot 在技术上是否同构、是否使用相同共识协议，以及差异边界。

---

## 2. 最终系统结构（落地形态）

### 2.1 网络层（虚拟 WAN）

- 核心编排文件：`docker-compose.routers.yml`
- 结构：
  - 10 个自治系统：AS201、AS202、AS203、AS204、AS205、AS206、AS207、AS208、AS209、AS210
  - 每个 AS 内部包含：
    - 边界路由器 BR（`10.2xx.0.254`）
    - 内部路由器 IR（`10.2xx.0.2` + `10.2xx.1.2`）
    - 交换节点（`10.2xx.1.3`）
  - IX 交换机 / Route Server：`ix-switch-10.250.0.2`
  - IX 网段：`10.250.0.0/24`

### 2.2 区块链节点层（Polkadot）

- 每个 AS 目标节点：
  - archive 1 个（固定 `.20`）
  - validator 6 个（目标段通常从 `.21` 开始动态分配）
  - full node 3 个（同样动态分配，避免冲突）
- 全网总节点：`10 × (1 + 6 + 3) = 100`
- 关键配置：`scripts/node_layout.yml`

### 2.3 监控与数据层（实验观测）

- 编排文件：`docker-compose.monitoring.yml`
- 组件：
  - PostgreSQL
  - Redis
  - Neo4j
  - `polkadot_node_cleaner`（集中清理与拓扑事件）
- 与节点侧监控代理配合用于实验数据采集。

## 4. 路由协议设计与实现细节

### 4.1 AS 内：OSPF（IR↔BR）

- IR 运行 BIRD OSPF，交换本 AS 内路由。
- BR 侧 BIRD 也与 IR 建立 OSPF 会话（`polkadot_bird.conf`）。

### 4.2 AS 间：BGP（BR↔IX Route Server）

- IX 运行 BIRD Route Server（ASN 100）；
- 各 AS BR 以 RS client 模式接入；
- RS 对各 AS 路由进行反射，实现多 AS 互通控制面。

## 7. 与真实 Polkadot 公链的技术同构性评估

### 7.1 已同构（协议与实现层）

1. **客户端实现同源**
  使用 `parity/polkadot` 客户端，网络层与共识执行逻辑来自同一代码体系。
2. **P2P 协议栈同类**
  节点通过 libp2p 栈进行点对点发现与通信，具备真实链网络行为特征。
3. **链规范来源同体系**
  当前链规格文件显示：
  - `name: Development`
  - `id: rococo_dev`
  - `protocolId: dot`  
   说明该实验链属于 Polkadot/Rococo 技术族谱内的开发链配置。
4. **共识机制语义同类（BABE + GRANDPA 体系）**
  在 Polkadot/Rococo Relay Chain 体系下，出块与终局采用 BABE/GRANDPA 组合：
  - BABE：按权重随机领导者选槽（可近似理解为 `P_i ≈ w_i / Σw`）。
  - GRANDPA：按权重投票完成最终性，安全阈值为超过 2/3 权重。  
   因此在“共识算法家族”层面，与公链是同构的。

### 7.2 未完全同构（生产治理与经济层）

1. **链身份不同**
  当前为 `rococo_dev/development` 实验链，不是 Polkadot 主网链 ID 与主网状态。
2. **验证者准入与密钥治理不同**
  实验环境通过目录与脚本生成节点与密钥，不等同于主网完整会话密钥管理流程。
3. **运行参数偏实验化**
  如 `--rpc-methods=unsafe`、`--force-authoring` 等参数用于实验便利，不是主网安全基线。
4. **经济安全与治理不等价**
  不具备主网真实 DOT 经济约束、去中心化治理流程与开放参与博弈环境。
5. **基础设施可靠性等级不同**
  Docker 单机/少量主机虚拟 WAN 的故障域、时钟源、带宽抖动与公网多地域主网不完全等价。

### 7.3 结论（严格表述）

- **结论 A（可以确认）**：该实验网络在“客户端实现 + 协议栈 + 共识机制家族”层面与 Polkadot 公链 **技术同构**。  
- **结论 B（必须保留边界）**：该网络不是 Polkadot 主网的 1:1 等价复制，属于 **同构的科研/工程实验链**，在治理、经济安全、运维安全基线方面与主网存在显著差异。
