# 结论

本文面向区块链网络拓扑优化问题，构建了基于网络演化动力学与自组织动力学的两阶段方法，并以 `unified_full_protocol` 和 `fair-budget` 为主文口径建立了真实平台优先的评估框架。结论章节仅接收已在 `claim_traceability.md` 标记为 `status=done` 的量化主张。

当前可提交版本结论（仅基于已 `done` 的主张）：

1. **Eth-Docker 图优化主证据（对应 C007）**  
在 `ED-MAIN-BASE` 与 `ED-MAIN-LINK/NETEM-{low,mid,high}` 共 7 个场景（`gradient_mode=full`、`n=10`）下，本文方法在 `R` 上相对四个基线均为正差分（fair 口径 `+0.05596~+0.19570`），在 `avg_path_len` 上均为负差分（`-0.70022~-0.67044`，多数比较 Holm 后显著）。`lcc_ratio` 差分整体较小且部分场景饱和，因此当前版本将其作为辅助指标而非主导优势证据。

2. **预算口径敏感性（对应 C003）**  
`fair-budget` 与 `raw-budget` 比较显示，ED 场景下 `R` 的 raw-fair 差异总体较小（多数在 `|0.02|` 以内）且不改变 OURS 相对基线的方向；因此主文固定 `fair-budget` 作为主排序口径，`raw-budget` 用于稳健性解释。

3. **可复现性与适用边界**  
本文通过 `comparison_id + stat_key + evidence_paths` 形成逐条证据链，所有已完成量化主张均可回溯到 `results/statistics/ED-MAIN-*/stats_summary_graph_optimizer.*` 与对应 manifest。跨平台结论（Eth-Cloud、BTC）在证据闭环前不进入确定性主结论；当前主文回填批次样本量为 `n=10`，后续若补跑 `n=20` 将按同一统计协议整章重算并覆盖。

综合而言，本文在已闭环的 ED 证据链上验证了“综合鲁棒性稳定提升 + 路径效率显著改善”的结果形态，并给出了可复核的统计与追溯协议。后续工作将聚焦于：  
- 在**本地部署 Eth-Cloud、BTC-Docker 平台栈**后，按同一协议补跑并产出 `results/statistics/` 中 EC/BTC 汇总，扩展跨协议栈主文证据；  
- 统一协议层指标以增强横向可比性；  
- 提升大规模攻击与长期演化场景的外推能力。
