# 实验设置

## 6.1 实验目标与口径

本章实验用于回答三类问题：  
（1）方法在主场景下是否显著优于主流基线；  
（2）结论在攻击扰动下是否稳健；  
（3）比较结论是否在公平预算下仍成立。  

主文仅使用 `unified_full_protocol`；主比较口径为 `fair-budget`。  
`raw-budget` 仅用于敏感性分析与解释，不用于主排序结论。

**审稿对齐说明**：（1）**基线全集**：主文对 `M-RESI/M-STAT/M-ASIM/M-FPS` 均须报告或指向附录全量表，禁止仅展示单一基线子集而不声明全集；（2）**失败与跳过**：若有 run 缺失或跳过，须在 manifest 或附录说明原因，禁止静默删除；（3）**主张分层**：观测（Layer A）、模型状态（Layer B）、统计推断（Layer C）不得混写语义（见仓库总领 `CLAUDE.md` 证据分层表）。

## 6.5 对比方法与公平性

主文基线：

1. **ResiNet**（学习式鲁棒重连）
2. **Static**（静态图启发式优化）
3. **AttackSim**（攻击仿真驱动优化）

补充对照：方法内部消融（仅演化、仅自组织、分项去除）。

公平性约束：

- 等时间预算或等边变更预算  
- 相同初始图、相同随机种子策略、相同停止准则  
- 基线提前收敛时记录真实开销，不强制耗尽预算  
- 预算映射规则必须在运行前固定，不得按结果回调参数

## 6.6 统计协议

- 报告：mean ± std, 95%CI  
- 关键对比：显著性检验 + 效应量  
- 多重比较：Holm 或 FDR（按实验矩阵预注册）  
- 主文阈值与假设：在实验配置中固定，不随结果变更  
- 正态性检查：Shapiro-Wilk（alpha=0.05）；正态用 paired t，非正态用 Wilcoxon  
- 95%CI：当前主线实现使用 t-interval（与 `experiments/tifs_stats.py::mean_std_ci95_t` 一致）；若启用 bootstrap 版本，需在实验矩阵与统计脚本中同步登记并整体重算
- 指标定义（方法对比）：`R.paired_delta = R_OURS - R_baseline`，其中 `R` 为 `R_s`、`R_c`、`R_r` 按固定权重组合的综合鲁棒性分数。  
- 解释口径：`R.paired_delta > 0` 表示 OURS 优于对应基线，`R.paired_delta < 0` 表示劣于对应基线；表中 `mean` 为跨 run 的均值，`p_value_holm` 为多重比较校正后的显著性结果。

## 6.6.1 目标-梯度对齐与记录

- 主线图优化默认采用 `gradient_mode=full`，以保证目标函数总 R 与梯度方向一致。  
- `R_s_only` 与 `R_s_R_r` 作为消融口径，必须与主线结果分表报告，不得混写。  
- 所有方法对比统计需可回溯到 `run_manifest_graph_optimizer.json` 中的 `gradient_mode` 字段。

## 6.6.2 统计族与稳健性补充（ED 主证据）

- Holm family（方法对比）：在同一 `experiment_id × budget_variant × metric` 下，`M-OUR` 对 `M-RESI/M-STAT/M-ASIM/M-FPS` 四个配对比较作为同一多重比较族。  
- FDR family（消融）：仅用于 `ablation_`* 变体，不与主文方法对比混族。  
- primary/secondary 约束：ED 主文方法对比以 `R.paired_delta` 为 primary，`lcc_ratio.paired_delta` 与 `avg_path_len.paired_delta` 为 secondary；主文允许 secondary 显著且 primary 存在 trade-off，但必须显式说明。  
- 稳健性检查：主文补充 1 个去极端 run（trim-1）敏感性测试，检验关键对比在删去绝对差分最大的单个 run 后，方向与显著性是否保持。对应产物见 `results/statistics/ED_robustness_sensitivity_trim1.csv`。

## 6.6.4 固定参数选取与冻结（主线）

- 主线方案仅使用固定权重 `w=(0.25,0.55,0.20)`；该配置与 `PROTOCOL_VERSION=tifs-freeze-v3` 绑定冻结。  
- 选取原则：  
  1. 外部公共指标优先（`lcc_ratio`、`avg_path_len`）；  
  2. 跨场景稳定优先（BASE/LINK/NETEM），不追单场景极值；  
  3. 显著性与效应量并行判定（`p_value(_holm)` + `effect_size`）。
- 在线自适应权重仅作为探索性/消融对照，不纳入主文方法定义与主结果排序口径。
