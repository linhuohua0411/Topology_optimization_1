# 相关工作

## 3.1 区块链拓扑优化

区块链拓扑优化研究大体可分为三类：静态属性驱动、攻击仿真驱动与图指标驱动。静态方法通常优化度分布、聚类系数或平均路径长度，工程可解释性较好，但对时变扰动的响应刻画不足。攻击仿真驱动方法强调抗毁评估，适于安全场景，但在高频仿真下计算开销较高。图指标驱动方法可快速给出结构改进方向，但难以统一局部连接机制与全局鲁棒性目标。

## 3.2 复杂网络鲁棒优化与学习方法

复杂网络鲁棒优化中，学习式重连方法（如 ResiNet 类工作）能够在给定预算内学习高质量重连策略，通常在标准网络数据集上表现突出。该路线的优势是搜索效率与表达能力，但在区块链场景常面临三项问题：  
（1）特征与奖励设计对场景迁移敏感；  
（2）预算口径不统一时比较不公平；  
（3）跨链栈（PoS/PoW/联盟链）可比性不足。  

## 3.3 网络演化与自组织动力学

网络演化模型（如 BA/WS）与自组织理论为拓扑时序建模提供了重要基础。已有研究多用于解释网络统计特性涌现，较少直接面向“可部署拓扑优化”问题。将演化探索与自组织收敛串联，并与预算约束和攻击评估协议联合，是本文与传统动力学研究的主要区别。

## 3.4 本文定位

与现有工作相比，本文定位在于：

1. 以动力学机制统一拓扑时序演化与鲁棒性优化目标；  
2. 以公平预算作为主文比较口径，避免“谁花更多代价谁更强”的偏差；  
3. 以真实平台可得数据为证据，在统一协议下构建可追溯实验链路：**本稿**优先闭环 Eth-Docker；Eth-Cloud、BTC-Docker 在对应平台栈于本地部署后按同一字段与统计口径补跑，以完成跨平台主文扩展。
