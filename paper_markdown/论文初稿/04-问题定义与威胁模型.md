# 问题定义与威胁模型

## 4.1 问题定义

本文研究对象为可观测、可评估、可约束的区块链网络拓扑（私有链/联盟链/受控云环境），目标是在攻击与故障条件下提升结构鲁棒性与传播相关性能，并输出可实施的拓扑调整建议。我们不假设在公链中可强制执行拓扑控制，公链语境下结论仅用于分析与建议。

## 4.2 Threat Model（五字段）

### attacker objective

攻击者目标是在给定预算下最大化网络退化：降低最大连通分量占比、拉长有效传播路径、增加传播不稳定性。

### attacker knowledge

攻击者可观测网络拓扑并在网络层实施结构扰动（节点/链路级）。不直接篡改共识协议逻辑与链上状态转换规则。

### attacker budget

预算由攻击强度与持续时间共同定义。主文采用结构扰动预算（节点移除比例或等价网络扰动强度）并在比较时执行公平预算对齐（时间上限、边变更上限）。

### attack modes

主文攻击模式（按平台能力映射；**何者已进入本版主文统计表**以 `results/statistics/` 与 `claim_traceability` 为准）：
- 节点级结构扰动：`random remove` / `targeted remove` / `adaptive remove`（Eth-Cloud、BTC；**本版 EC/BTC 统计待本地补跑**）  
- 链路级与网络条件扰动：`link down`、`tc netem`（Eth-Docker；**本版已部分闭环**）  
- 网络分区与路由扰动（BTC；**本版以扩展实验形式已回填 `mid` 档画像**；全强度与方法对比仍属可选补强；其他平台按可得性进入附录）

附录扩展仅用于未进入主文统计闭环的平台或未达样本门槛的变体，不得用于主文量化主张。

### evaluation protocol

（1）主文口径固定为 `unified_full_protocol`。  
（2）主比较口径固定为 `fair-budget`，`raw-budget` 仅做补充解释。  
（3）关键指标统一报告 mean±std、95%CI、显著性检验与效应量。  
（4）路径指标统一在最大连通分量内统计，LCC 统一按“攻击后最大连通分量节点数 / 原始节点数”定义。  
（5）仅将“字段稳定可采”的指标纳入主文关键结论；不稳定字段仅用于补充分析。

## 4.4 非目标与边界

- 不将主文结论外推为“公网可直接部署效果”。  
- 不将可微代理目标等同于完整链上安全语义。  
- 对于未稳定可采的接口，不作为主文关键结论依据。  
- Pol-Docker 当前运行态未确认，不进入本轮主文定量对比。
